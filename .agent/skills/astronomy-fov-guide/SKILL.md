# Astronomy FOV & Orientation Guide

在星图软件开发和天文摄影中，准确区分不同的角度定义对于正确绘制相机视场（FOV）至关重要。

## 1. 核心术语定义与几何关系

### 1. Position Angle (PA) - 位置角
*   **定义**：目标（或相机垂直轴/长边 X 轴）相对于**天北极 (Celestial North)** 的旋转角度。
*   **测量方式**：从北向东（逆时针）旋转。
    *   0° 指向北
    *   90° 指向东
*   **应用**：星图软件中最通用的标准。
    *   若 $PA = 90^\circ$，相机长边（通常是 X 轴）将平行于赤纬线，短边指向北。
*   **基准线**：天北极 (固定坐标)。

### 2. Rotation Angle (° E of N) - 旋转角（北偏东）
*   **定义**：通常是 Position Angle 的另一种表达方式。
*   **关系**：在大多数现代摄影软件（如 N.I.N.A.）中，它指的就是相机坐标系相对于天球坐标系的偏移。
*   **注意**：硬件相关。某些机械旋转器（Rotator）会有自己的“机械零点”，其读数可能与天球 PA 有一个恒定的偏移量，开发时需注意校准。

### 3. Polar Angle - 极角
*   **定义**：极坐标系中的通称。在天文软件中，有时用来描述视场长边与指向天极方向之间的夹角。
*   **特定场景**：“长边与天极连线垂直” 对应 $PA = 90^\circ$ 或 $270^\circ$。

### 4. Parallactic Angle - 旁轴角 / 视差角
*   **定义**：**天球北极**、**目标物体**、**当地天顶 (Zenith)** 三点构成的夹角。
*   **区别**：
    *   **PA** 是相对于“北”的，是固定的天球坐标。
    *   **Parallactic Angle** 是相对于“头顶”的（地平坐标系）。随着地球自转，目标在天空中的位置改变，这个角会**实时变化**。
*   **应用**：主要用于计算**大气折射**的方向。赤道仪用户通常不需要手动调节它，但地平仪（Alt-Az Mount）如果不通过旋转器抵消此角度，拍摄会出现场旋。

## 2. 核心区别总结表

| 术语 | 基准线 | 变化情况 | 绘图/开发用途 |
| :--- | :--- | :--- | :--- |
| **Position Angle (PA)** | 天北极 | 构图固定后不变 | **定义相机 FOV 框的旋转** (构图标准) |
| **Parallactic Angle** | 天顶 (垂直地面) | 随时间实时变化 | 计算大气色散、地平仪场旋修正 |
| **Rotation Angle** | 相机初始位 | 取决于驱动程序 | 读取机械旋转器数值 |

## 3. 在星图中绘制 FOV 的实现策略

在开发中，应区分“计划构图”与“实际指向”两种不同的 FOV 显示模式。

### 模式 A: 目标 FOV (Centered Camera FOV)
**用途**：用于构图，决定如何旋转相机才能把目标（如星云）放进画面。
*   **中心点 (Center)**：目标星体的 RA/Dec 坐标。
*   **旋转角 (Rotation)**：使用用户输入的 **Position Angle (PA)**。
*   **逻辑**：以天北极为 0° 基准。

### 模式 B: 当前指向 FOV (Mount Current FOV)
**用途**：显示设备当前的实际指向和旋转状态。
*   **中心点 (Center)**：从赤道仪 (Mount) 实时回传的 RA/Dec 坐标 (ASCOM/INDI)。
*   **旋转角 (Rotation)**：相机的**物理 PA**。
    *   *数据来源*：通常通过 Plate Solving (盲解/解析) 获得的解算坐标，或机械旋转器的读数（需校准）。
*   **现象**：如果望远镜没有精确对齐 (Sync)，此框的中心可能会偏离目标星体。

## 4. 开发建议
*   在绘制 FOV 矩形时，始终明确当前的旋转基准是 **Celestial North** 还是 **Zenith**。
*   对于赤道仪系统，几乎总是使用 **PA** (相对于北极)。
*   仅在处理地平式望远镜的场旋模拟或大气色散校正工具时，才引入 **Parallactic Angle**。
